## 11\. Isi Wajib `REKAP\_PENGEMBANGAN.md`

Log historis, format entri per tanggal/versi, ditulis setiap kali ada perubahan (fitur baru, update, atau penghapusan). Format standar tiap entri:

```markdown
## \[YYYY-MM-DD] — v0.x.x

### Ditambahkan
- ...

### Diubah
- ...

### Dihapus
- ...

### Catatan Teknis
- (mis. alasan keputusan arsitektur, breaking change, migrasi data yang diperlukan)
```

Entri terbaru selalu ditaruh paling atas (reverse chronological). File ini menjadi sumber kebenaran untuk tracking progres pengembangan lintas sesi/lintas developer.
