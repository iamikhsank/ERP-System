## 7\. Build \& Deployment Workflow

1. **Development Lokal:** Jalankan `npm install` → `npm run dev`. Gunakan mock data, karena `google.script.run` tidak berjalan di luar ekosistem GAS (sediakan mock layer di `gasClient.js`).
2. **Backend Modular:** Development backend wajib dilakukan secara modular di dalam folder `gas-src/` (per modul), **bukan mengedit langsung di file gabungan**.
3. **Build Backend (`npm run build:gas`):** Menjalankan script `scripts/build-gas.mjs` yang akan:

   * Menyalin `gas-src/00-setup.gs` menjadi `dist-gas/setup.gs` (tanpa modifikasi).
   * Menggabungkan (concat) `01-main.gs` + `02-auth.gs` + `modules/\*.gs` (berdasarkan urutan prefix angka) + `99-utils.gs` menjadi `dist-gas/code.gs`, lengkap dengan header komentar penanda asal tiap blok kode.
4. **Build Frontend (`npm run build`):** Menjalankan Vite build menggunakan `vite-plugin-singlefile` untuk menghasilkan 1 file HTML di folder `dist/`, kemudian disalin/di-rename menjadi `webapp.html`.
5. **Build Keseluruhan (`npm run build:all`):** (Direkomendasikan) Script gabungan untuk mengeksekusi langkah 3 \& 4 secara bersamaan.
6. **Push ke Apps Script:** Salin `dist/webapp.html`, `dist-gas/code.gs`, dan `dist-gas/setup.gs` ke project Apps Script. (Sangat direkomendasikan menggunakan `clasp push` untuk standar enterprise agar versioning Git terjaga; arahkan `rootDir` di `.clasp.json` ke folder output tersebut).
7. **Inisialisasi Database:** Jalankan fungsi `runSetup()` satu kali secara manual dari editor GAS untuk melakukan inisialisasi skema Sheets.
8. **Deployment Web App:** Deploy project sebagai **Web App** (Konfigurasi: `Execute as: Me`, `Who has access: sesuai kebutuhan organisasi`).
9. **Automasi CI/CD:** Manfaatkan `clasp`, `.clasp.json`, dan `appsscript.json` (manifest: set `webapp.executeAs` \& `access`) agar proses deployment ke GAS dapat diotomatisasi melalui workflow `.yml` (serupa dengan `deploy-pages.yml` di 4.3, namun dengan target Apps Script).

> \*\*Ringkasan package.json scripts:\*\* `dev` (lokal), `build` (frontend/Vite), `build:gas` (backend concat), `build:all` (frontend + backend), `build:pages` (mode preview untuk GitHub Pages).
