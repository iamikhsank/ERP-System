## 10\. Isi Wajib `REQUIREMENTS.md`

File ini adalah rujukan teknis tunggal — setiap keputusan konfigurasi harus tercatat di sini, minimal berisi:

1. **Daftar Library \& Versi CDN**

   * Nama library, versi yang dipin, URL CDN lengkap (jsDelivr/esm.sh), fungsinya di app, dan alasan pemilihan (kenapa bukan alternatif lain).
   * Contoh format tabel: `| Library | Versi | CDN URL | Dipakai di | Alasan |`
2. **Import Map lengkap** — salinan persis isi `<script type="importmap">` yang dipakai di production.
3. **Konfigurasi Vite (`vite.config.js`)** — dijelaskan per opsi: plugin yang dipakai (`vite-plugin-singlefile`, dsb), `build.rollupOptions.external` (module apa saja yang di-exclude dari bundle karena sudah lewat CDN), base path, mode (`development`/`preview`/`production`) beserta perbedaan tiap mode.
4. **Konfigurasi file lain**: `appsscript.json` (manifest GAS — timezone, exceptionLogging, webapp access), `.clasp.json`, `package.json` (scripts: `dev`, `build`, `build:gas`, `build:all`, `build:pages`), dan `scripts/build-gas.mjs` (logika concat file `gas-src/\*.gs` → `dist-gas/code.gs`, termasuk urutan file \& format komentar penanda blok).
5. **Font** — nama font (heading \& body), sumber (Google Fonts CDN), fallback stack, ukuran dasar (base font-size, scale ratio untuk heading).
6. **Palet Warna** — daftar lengkap dengan kode HEX/HSL untuk: primary, secondary, accent, background (light/dark), surface/card, border, serta warna semantik (success/warning/error/info). Dijelaskan juga di halaman/komponen mana tiap warna dipakai.
7. **Design tokens lain**: spacing scale, border-radius scale, shadow levels — merujuk ke `styles/tokens.css`.
