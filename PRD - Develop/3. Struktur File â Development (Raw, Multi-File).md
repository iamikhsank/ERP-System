## 3\. Struktur File — Development (Raw, Multi-File)\[cite: 2]

Struktur ini untuk tahap development di lokal (bukan yang diupload ke Apps Script):\[cite: 2]

```text
/erp-app
├── src/                                # SOURCE CODE FRONTEND (REACT + VITE + TAILWIND)
│   ├── main.tsx                        # Entry point utama React, menginisialisasi DOM & Router
│   ├── App.tsx                         # Layout Wrapper utama: penanganan Sidebar navigasi, Topbar profil, responsivitas menu, & Error Boundary global
│   ├── types.ts                        # Kontrak tipe data & interface TypeScript terpusat untuk seluruh entitas ERP (Inventory, Finance, HR, Procurement, Sales)
│   ├── index.css                       # Stylesheet global yang mengintegrasikan framework Tailwind CSS (@import "tailwindcss")
│   ├── api/
│   │   └── gasClient.ts                # Jembatan asinkron komunikasi frontend-backend.
│   │                                     - Mode PROD: Memanggil API google.script.run secara asinkron (Promise-based)
│   │                                     - Mode DEV: Menyediakan simulasi persistensi lokal (localStorage 'erp_mock_data') lengkap dengan kalkulator metrik real-time & automasi akuntansi berantai
│   ├── components/                     # REUSABLE UI COMPONENTS (Premium UI Elements)
│   │   ├── DataTable.tsx               # Komponen tabel data dinamis yang dilengkapi dengan fitur pencarian teks, filter kategori cerdas, sorting kolom, dan sistem penomoran halaman (pagination)
│   │   ├── Modal.tsx                   # Komponen dialog popup interaktif untuk formulir input tambah dan edit data
│   │   ├── Toast.tsx                   # Sistem notifikasi melayang (pop-up toast) untuk memberikan feedback sukses/gagal secara elegan kepada pengguna
│   │   └── ErrorBoundary.tsx           # Proteksi tingkat komponen untuk mengisolasi error runtime di client agar aplikasi tidak mati total
│   └── pages/                          # ELEMEN HALAMAN / MODUL ERP UTAMA
│       ├── Dashboard.tsx               # Halaman ringkasan analitik, grafik arus kas (Recharts), pencapaian KPI bisnis, dan log aktivitas real-time
│       ├── Inventory.tsx               # Modul pengelolaan stok persediaan barang, multi-gudang, dan visualisasi status stok kritis (low stock alert)
│       ├── Finance.tsx                 # Modul pencatatan kas keuangan perusahaan (Pemasukan & Pengeluaran) lengkap dengan pengelompokan kategori transaksi
│       ├── HR.tsx                      # Modul kepegawaian untuk pengelolaan data karyawan, posisi, status kontrak kerja, dan slip payroll bulanan
│       ├── Procurement.tsx             # Modul sistem pengajuan pembelian/pengadaan barang lengkap dengan sistem persetujuan bertingkat (Approval Flow)
│       ├── Sales.tsx                   # Modul pencatatan pesanan penjualan (Sales Order), data pelanggan, status invoice, dan pemicu kas masuk otomatis
│       ├── Reporting.tsx               # Modul rekapitulasi pelaporan komparatif lintas departemen
│       └── Settings.tsx                # Modul pengaturan konfigurasi sistem global (Profil Perusahaan, timezone, format penomoran otomatis)
│
├── gas-src/                            # SOURCE CODE BACKEND MODULAR (Satu-satunya Sumber Kebenaran Backend)
│   ├── 00-setup.gs                     # Skrip idempotent penyiapan awal spreadsheet: otomatis membuat sheet, kolom header, seeding user admin pertama, dan Script Properties
│   ├── 01-main.gs                      # Skrip inisialisasi Web App: berisi fungsi doGet(e) untuk merender file HTML produksi tanpa sandbox usang demi menghindari crash runtime
│   ├── 02-auth.gs                      # Manajemen autentikasi session pengguna aktif, otorisasi otentik, dan validasi izin akses berdasarkan peran (Role-Based Access)
│   ├── 99-utils.gs                     # Kumpulan fungsi pustaka pembantu backend (API response wrapper format { success, data, error }, logger internal)
│   └── modules/                        # BACKEND CONTROLLERS (API Endpoints per Modul)
│       ├── 10-dashboard.gs             # Logika penarikan agregasi data, kalkulator performa, dan log aktivitas gabungan di Google Sheets
│       ├── 11-inventory.gs             # API CRUD untuk mengelola data barang, pembaruan stok, dan warning batas minimum persediaan
│       ├── 12-finance.gs               # API CRUD pencatatan kas masuk & keluar langsung pada Google Sheets Keuangan
│       ├── 13-hr.gs                    # API CRUD manajemen profil karyawan, pembaruan status kerja, dan kalkulasi slip payroll
│       ├── 14-procurement.gs           # API CRUD pengadaan barang serta penanganan persetujuan (approve/reject) pengajuan
│       ├── 15-sales.gs                 # API CRUD pembuatan order penjualan dan pemicu transaksi keuangan otomatis (chained automation)
│       ├── 16-reporting.gs             # API untuk kompilasi dan ekstraksi laporan keuangan & inventori berkala
│       └── 17-settings.gs              # API untuk pembaruan pengaturan perusahaan secara persistent di Script Properties
│
├── scripts/                            # SCRIPTS OTOMATISASI DAN BUILD PROCESS
│   ├── build-gas-backend.mjs           # Skrip Node.js otomatis penggabung (concat) file-file modular di gas-src menjadi file tunggal 'dist-gas/code.gs' sesuai urutan numerik prefix
│   └── build-gas-frontend.mjs          # Skrip Node.js pasca-build Vite yang memotong spasi kosong redundan di 'dist/index.html' dan menyalinnya menjadi file tunggal 'Dashboard-for-Spreadsheet.html' di root
│
├── dist-gas/                           # OUTPUT BACKEND SIAP DEPLOY (Auto-generated, Dilarang Edit Manual)
│   ├── setup.gs                        # Salinan langsung dari 00-setup.gs untuk diunggah ke editor GAS
│   └── code.gs                         # Gabungan seluruh backend controller modular siap pakai yang terbebas dari tabrakan scope global
│
├── Dashboard-for-Spreadsheet.html      # OUTPUT FRONTEND SIAP DEPLOY (Auto-generated di Root)
│                                         - File HTML tunggal (single-file HTML) hasil build ter-inline & ter-minify sepenuhnya dari Vite
│                                         - Inilah file yang wajib diunggah ke Editor Google Apps Script
│
├── index.html                          # Template HTML utama yang digunakan oleh Vite Dev Server untuk pengembangan lokal
├── package.json                        # Definisi dependency project, konfigurasi lint, serta daftar skrip build terintegrasi (dev, build:html, build:gas, build:all)
├── REQUIREMENTS.md                     # Dokumentasi spesifikasi library pihak ketiga, tautan CDN resmi, aturan Vite, dan pedoman resolusi bug
└── REKAP_PENGEMBANGAN.md               # Log histori lengkap setiap rilis, fitur baru, perbaikan, dan perubahan struktur data (versioning control)
```

> \*\*3 file dokumentasi di atas (`README.md`, `REQUIREMENTS.md`, `REKAP\_PENGEMBANGAN.md`) WAJIB ada di root project dan wajib selalu di-update setiap ada perubahan kode.\*\*\[cite: 2] Detail isi masing-masing ada di bagian 10–12.\[cite: 2]

> \*\*Kenapa `gas-src/` tetap modular padahal target akhirnya 3 file?\*\*\[cite: 2] Google Apps Script sebenarnya otomatis menggabungkan semua file `.gs` dalam satu project ke satu global scope saat runtime — jadi secara teknis file backend \*\*boleh\*\* tetap terpisah per modul di Apps Script Editor dan tetap jalan normal.\[cite: 2] Tapi karena requirement project ini \*\*strict 3 file total\*\* (`setup.gs`, `code.gs`, `webapp.html`), penulisan kode backend \*\*tetap dilakukan modular per file\*\* (mirip cara frontend dipecah per komponen), lalu \*\*digabung otomatis lewat build script\*\* (`scripts/build-gas.mjs`) — bukan ditulis manual di satu file besar.\[cite: 2] Ini menjaga kemudahan maintain \& AI-generation per modul, sekaligus tetap memenuhi constraint 3 file saat deploy.\[cite: 2]
