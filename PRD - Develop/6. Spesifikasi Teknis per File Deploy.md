## 6\. Spesifikasi Teknis per File Deploy

### `setup.gs`

* **Fungsi `runSetup()`:** Melakukan pengecekan dan membuat Spreadsheet aktif sebagai database jika belum ada sheet yang dibutuhkan.
* **Pembuatan Sheet:** Membuat sheet per modul dengan header baku (kolom `id`, `createdAt`, `updatedAt`, `createdBy`, dst + kolom spesifik modul).
* **Manajemen Akses Awal:** Membuat sheet `Users` dan `Roles` serta melakukan seeding 1 admin default.
* **Konfigurasi Sistem:** Mengatur `PropertiesService.getScriptProperties()` untuk konfigurasi global seperti nama perusahaan, timezone, dan format nomor dokumen.
* **Idempotent:** Script bersifat aman untuk dijalankan berulang kali tanpa risiko duplikasi data.

### `code.gs` (Hasil build, sumber asli di `gas-src/01-main.gs`, `02-auth.gs`, `modules/\*.gs`, `99-utils.gs`)

* **Fungsi `doGet(e)` (di `01-main.gs`):** Melakukan validasi session user, merender `webapp.html`, serta meng-inject data awal seperti informasi user dan role via templated HTML jika diperlukan.
* **Fungsi API Modul:** Setiap modul (`modules/11-inventory.gs`, `12-finance.gs`, dst) berisi fungsi API dengan pola konsisten yaitu `getX()`, `createX(data)`, `updateX(id, data)`, dan `deleteX(id)`. Setiap fungsi wajib dilengkapi dengan validasi input dan pengecekan permission, serta **ditulis \& di-review per file modul**, bukan langsung di file gabungan.
* **Helper Otentikasi (di `02-auth.gs`):** Menyediakan fungsi `getCurrentUser()` dan `checkPermission(module, action)`.
* **Standardisasi Error (di `99-utils.gs`):** Mengembalikan objek dengan format seragam `{ success: boolean, data, error }` agar konsisten dikonsumsi oleh frontend.
* **Build Automation (`scripts/build-gas.mjs`):** Menggabungkan (concat) semua file di atas sesuai urutan prefix angka (`01-`, `02-`, `10-…17-`, `99-`) menjadi satu file tunggal `dist-gas/code.gs`. Masing-masing blok diberi komentar penanda asal file agar tetap mudah ditelusuri atau di-debug meskipun sudah disatukan.

### `webapp.html`

* **Output Vite:** Merupakan hasil build otomatis dari Vite menggunakan `vite-plugin-singlefile` dan tidak ditulis secara manual.
* **Struktur Internal:** Sebelum di-build, struktur frontend mengikuti pola SPA di folder `src/` dengan router berbasis hash-based dan mekanisme lazy render per halaman ke dalam kontainer `<div id="app">`.
* **Asynchronous Bridge:** Wajib menggunakan sintaks `google.script.run.withSuccessHandler().withFailureHandler()` yang dibungkus di dalam Promise pada file `gasClient.js` agar kode async di frontend menjadi lebih rapi (`await callGas('getInventory')`).
