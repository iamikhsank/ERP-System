## 12\. Isi Wajib `README.md`

Dokumentasi paling detail, ditujukan untuk developer baru yang belum tahu apa-apa tentang project ini. Minimal mencakup:

1. **Deskripsi Proyek** — apa itu, untuk siapa, masalah apa yang diselesaikan.
2. **Arsitektur** — diagram/penjelasan alur data (Frontend ↔ `google.script.run` ↔ `code.gs` ↔ Google Sheets), termasuk penjelasan kenapa pakai CDN + import map.
3. **Struktur Folder** — penjelasan tiap folder/file penting (boleh merujuk bagian 3 \& 4 dokumen ini).
4. **Cara Setup Lokal** — clone, `npm install`, menjalankan `npm run dev`, cara pakai mock data GAS.
5. **Cara Build \& Deploy ke Apps Script** — step-by-step lengkap termasuk `clasp login`, `clasp push`, cara menjalankan `runSetup()`, cara deploy sebagai Web App (beserta screenshot/placeholder instruksi klik).
6. **Cara Deploy Preview ke GitHub Pages** — menjelaskan workflow `.yml`, cara trigger manual (`workflow\_dispatch`), dan batasan preview (data mock, bukan data real).
7. **Daftar Modul \& Fitur** — ringkasan tiap modul ERP beserta fitur utamanya.
8. **Role \& Permission** — tabel role vs akses modul.
9. **Troubleshooting** — masalah umum (mis. quota GAS exceeded, CORS saat load CDN, import map tidak ke-resolve di browser lama) beserta solusinya.
10. **Kontribusi** — konvensi commit, cara update `REKAP\_PENGEMBANGAN.md` setiap PR/perubahan, coding style.
