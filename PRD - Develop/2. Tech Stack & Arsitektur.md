## 2\. Tech Stack \& Arsitektur\[cite: 2]

|Layer|Teknologi|
|-|-|
|Backend / Server|Google Apps Script (`.gs`, V8 runtime)\[cite: 2]|
|Database|Google Sheets (per-modul sheet/tab, dengan skema kolom terstandar)\[cite: 2]|
|Frontend Dev|HTML5 + CSS3 (Tailwind via CDN) + Vanilla JS / lightweight lib, dibangun dengan **Vite**\[cite: 2]|
|Library Loading|**Wajib via CDN** (bukan bundled npm) untuk semua library besar\[cite: 2]|
|Module Resolution|**Import Map** (native browser ES module resolution ke CDN)\[cite: 2]|
|Build Output|Single-file HTML (hanya kode aplikasi ter-inline; library tetap load dari CDN) via `vite-plugin-singlefile`\[cite: 2]|
|Auth|Google OAuth (Session.getActiveUser) + custom role table di Sheets\[cite: 2]|
|Komunikasi FE↔BE|`google.script.run` (async call ke Apps Script functions)\[cite: 2]|
|Error Handling|Global error boundary di frontend + console panel custom\[cite: 2]|
|Deployment (App)|Apps Script Web App deployment (`doGet`)\[cite: 2]|
|Deployment (Preview)|GitHub Pages via GitHub Actions (`.yml`) — untuk preview UI statis\[cite: 2]|

**Prinsip arsitektur:**\[cite: 2]

* **REST-like API Layer**: Backend GAS berperan sebagai REST-like API layer + auth gate, bukan sekadar script tempel\[cite: 2]. Semua akses data lewat fungsi terpusat di `code.gs` (tidak ada akses langsung dari client ke Sheets)\[cite: 2].
* **SPA Routing**: Frontend adalah SPA (Single Page Application) dengan routing berbasis hash (`#dashboard`, `#inventory`, dst) karena GAS `doGet` hanya render satu entry point\[cite: 2].
* **Lightweight Build**: File tetap kecil \& ringan karena library pihak ketiga TIDAK di-bundle ke dalam `webapp.html`\[cite: 2]. Semua library (chart, utility, dsb) di-load via `<script type="module">` + **import map** yang menunjuk ke CDN (jsDelivr/unpkg/esm.sh)\[cite: 2]. Vite hanya membundle kode aplikasi milik sendiri (business logic, komponen, router)\[cite: 2].
