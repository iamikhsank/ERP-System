## 1\. Pendahuluan \& Konsep Utama

Sistem ERP ini dibangun sepenuhnya di atas ekosistem Google Workspace untuk menjawab kebutuhan bisnis akan platform operasional yang terintegrasi, hemat biaya, dan *enterprise-ready*.

**Objektif Utama**
Mengembangkan aplikasi web ERP modern, cepat, dan aman menggunakan infrastruktur Google, dengan tetap mempertahankan standar *software engineering* profesional (modularitas, arsitektur data yang baik, dan kemudahan *maintenance*).

**Konsep Arsitektur Inti**

* **Backend \& Hosting:** Sepenuhnya dikelola oleh infrastruktur Google Apps Script (GAS).
* **Database:** Menggunakan Google Sheets dengan struktur tabel (sheet) terstandar layaknya RDBMS.
* **Frontend:** Dikembangkan menggunakan *tooling* modern berbasis Vite, yang kemudian di-build menjadi satu file HTML tunggal (`webapp.html`) sesuai dengan requirement *HtmlService* GAS.
* **Dual Output Workflow:** Memisahkan lingkungan kerja menjadi dua wujud paralel: versi *development* (kode raw, multi-file, terstruktur per modul) dan versi *production* (file gabungan/terkompresi hasil build yang siap di-deploy ke Apps Script).

**Profil Pengguna \& Hierarki**
Sistem ini dirancang untuk mengakomodasi berbagai tingkatan pengguna organisasi (Admin, Manager, Staff, Viewer) yang diatur menggunakan *Role-Based Access Control* (RBAC) ketat yang tervalidasi langsung di sisi server (backend).

**Batasan Kuota GAS \& Mitigasi Teknis**
Mengingat Google Apps Script memiliki limitasi performa (*cold start*) dan batasan kuota eksekusi harian:

* **Mitigasi Latensi:** Menerapkan pola *Single Page Application* (SPA) dengan *lazy render* serta *skeleton loading state* di UI. Komunikasi data ke backend dikumpulkan secara asinkron untuk menekan *round-trip* yang tidak perlu.
* **Mitigasi Payload/Ukuran File:** Menghindari *bundling* library besar ke dalam aplikasi. Semua *library* pihak ketiga wajib dimuat melalui mekanisme CDN dan *Import Maps* agar file `webapp.html` tetap ringan dan tidak memberatkan *memory limit* eksekusi *script*.
